# Changelog

## [1.0.0-rc.4] - unreleased

### Fixed
- **Unconditional `apply { ... }` with `$param` expressions froze at the constructor value** ŌĆö `apply { scale: $k; alpha: $a; ... }` without a `@()` wrapper baked the expressions once at build/construction time and never re-evaluated. `setParameter("k", 2.0)` left `scaleX` stuck at the initial value in both the runtime builder (incremental) and codegen paths. Conditional apply was unaffected because it flows through `_applyVisibility()` / `applyConditionalChains()` which already re-evaluate on param change. Fix: codegen `processApply` now registers each property update (scale/rotation/alpha/tint/filter/position) in `expressionUpdates` with collected `paramRefs`, mirroring the element path; builder drops the `node.conditionals != NoConditional` guard in the APPLY case so unconditional entries are also tracked via `captureApplyBaseline` + `trackConditionalApply`, and `resolveVisibilityForChildren` NoConditional branch calls `reconcileApplyParent` so the per-parent baseline-replay machinery re-fires. Two regression tests in `ApplyParamUpdateTest` (builder + codegen) backed by fixture `test/examples/108-applyParamUpdate`.
- **Interleaved `@(cond) apply { ... }` on the same parent corrupted state on unapply** ŌĆö when two or more conditional `apply` blocks modified the same parent property (alpha/scale/rotation/filter/position) and their conditions could both be active, unapplying the outer while the inner stayed matched left the parent in the pre-any-apply state instead of the still-matched inner's value. Root cause: per-entry `savedAlpha`/etc. captured whatever the parent's property was *at the moment that entry applied* ŌĆö which already included any earlier apply's mutation. Fixed by replacing per-entry save/restore with a per-parent baseline (`IncrementalUpdateContext.conditionalApplyBaselines`) captured lazily before the first conditional apply touches that parent. Any apply/unapply on that parent resets to baseline and replays all currently-matched entries in declaration order (`reconcileApplyParent`), so overlapping applies compose correctly and decouple from the order in which conditions flip. Three regression tests in `BuilderUnitTest`: `testIncrementalApplyAlphaInterleavedSameProperty`, `testIncrementalApplyScaleInterleavedSameProperty`, `testIncrementalApplyAlphaInterleavedStartHidden`.
- **`repeatable`/`repeatable2d` loop variable could silently shadow a programmable parameter** ŌĆö the parser accepted `repeatable($paramName, ...)` where `$paramName` was already a parameter of the enclosing programmable. `indexedParams` is shared between parameters and loop vars, so inside the body `$paramName` resolved to the loop index (int) instead of the parameter; worse, after the loop `MultiAnimBuilder.build` runs `indexedParams.remove(varName)` unconditionally, permanently erasing the parameter for any subsequent references in the same programmable. Parser now rejects: loop vars that match a parameter (`repeatable` + `repeatable2d`), loop vars that match any outer `scopeVars` entry (nested loops / iterator outputs / `@final`), `repeatable2d` with identical X and Y loop vars, iterator output vars (`array($v, ŌĆ”)`, `stateanim($b, ŌĆ”)`, `tiles($b, $t, ŌĆ”)`) matching a parameter or the main loop var, and the two `tiles(...)` outputs matching each other. One pre-existing `.manim` collision in `proto-game/res/manim/cards.manim` (`repeatable($cost, step($cost, dx:-12))` inside `#escDetailCard programmable(ŌĆ” cost:0..5=3 ŌĆ”)`) was updated to `repeatable($pip, step($cost, dx:-12))`; visible output is unchanged because the body never referenced `$cost`. Five new parser error tests in `ParserErrorTest`.
- **`@else`/`@default` chain after a terminal (bare) `@else` or `@default`** ŌĆö conditional chains used to accept `@else` / `@default` following a bare `@else` or `@default`, even though those earlier arms are unconditional and make every trailing arm unreachable. Parser now rejects this at parse time with a clear "cannot follow a terminal @else or @default" message. Four new tests in `ParserErrorTest` (`testElseAfterBareElse`, `testDefaultAfterBareElse`, `testElseAfterDefault`, `testDefaultAfterDefault`). `test/examples/107-rubeGoldberg/rubeGoldberg.manim` restructured to make its `@default` start a fresh chain via `@(count >= 5) ŌĆ” @default ŌĆ”`.

### Added
- **`BuilderError` typed runtime exception + structured error surfaces** ŌĆö new `bh.multianim.BuilderError extends haxe.Exception` carries the offending `Node` (for source position via `parsedPos():{file, line, col}`) and an optional `code:String` tag for programmatic filtering. Three call shapes: (1) `builderError(msg, ?code)` member helper on `MultiAnimBuilder` ŌĆö auto-attaches `currentNode`; (2) `builderErrorAt(node, msg, ?code)` member helper ŌĆö for sites where an explicit node is in scope (replaces `throw '...' + MacroUtils.nodePos(node)`); (3) `BuilderError.of(msg, ?code)` static factory ŌĆö for helper classes (`BuilderResult`, `SlotHandle`, `BuilderResolvedSettings`) that have no `currentNode`. All 262 throws in `MultiAnimBuilder.hx` migrated (was 256 plain-string throws + 6 added by Phase 1/2/3 work). Error-surface catch sites in `ScreenManager.rebuildAll()`, `ScreenManager.makeHotReloadFailError()` (powers DevBridge `hot_reload` + SSE notifications), `ScreenManager.strictFail()` (under `-D MULTIANIM_STRICT`), and `DevBridge.handleEvalManim()` (MCP `eval_manim`) now branch on `BuilderError` and return `{file, line, col, code}` to clients. MCP consumers can now render clickable diagnostics for runtime `.manim` errors. Established codes: `"not_a_number"` (expected numeric but got string/color/callback-string), `"missing_ref"` (reference/array/animation not found). Consumer-side `catch (e)` / `catch (e:Dynamic)` / `catch (e:haxe.Exception)` still match (via `haxe.Exception` inheritance); `'$e'` formatting preserved through `toString()` override. **Sibling runtime builder files migrated**: `ProgrammableBuilder.hx` (11 throws; particle/tileGroup lookups attach `progNode` for parserPos), `layouts/MultiAnimLayouts.hx` (6 throws), `ProgrammableUpdatable.hx` (4 throws) ŌĆö all now route through `BuilderError.of` (or `new BuilderError(msg, node)` where a node is in scope), so their errors also reach the structured-diagnostic path. Parse-time lexer errors in `MacroManimParser.hx` and runtime-injected throws emitted by `ProgrammableCodeGen.hx` into user code remain plain strings (out of scope ŌĆö they run before or outside the builder error surface).
- **`HeapsUtils.solidTile` / `solidBitmap` helpers** ŌĆö single shared path for solid-color `h2d.Tile` / `h2d.Bitmap` creation. `solidTile(color, w, h)` returns a 1├Ś1 GPU texture stretched to `(w, h)`; `solidBitmap(color, w, h, ?parent)` is the Bitmap convenience. Color is strict-D `0xAARRGGBB` ŌĆö top byte is alpha (e.g. `0x80FF8000` = 50%-alpha orange). Top-byte=0 is treated as opaque to match Heaps' `Tile.fromColor` default and keep bare `0xRRGGBB` callers working; for fully transparent, call `Tile.fromColor` directly. Migrated call sites: `MAObject` debug hitbox overlays, `MultiAnimBuilder.generatePlaceholderBitmap` (`SolidColor` generated tile), text tile background, the incremental `SolidColor` update path, `ScreenManager` modal overlay, and the `ProgrammableCodeGen` `SolidColor`/`Cross` emitters (the legacy inline `if (c >>> 24 == 0) c |= 0xFF000000` bake ŌĆö a no-op once Heaps masks the top byte ŌĆö is deleted).
- **DevBridge `debugger()` API and `get_debugger_hits` RPC** ŌĆö JS-`debugger;`-style breakpoint for game code. `screenManager.devBridge.debugger(data, pause=true)` snapshots `data`, auto-captures file/line/method via `haxe.PosInfos`, pushes an SSE `debugger` event, appends the hit to a ring buffer (capacity 100), and optionally pauses the game loop (shared `setPaused()` helper extracted from `handlePause`). Poll via new `get_debugger_hits` RPC (params: `clear`, `limit` Ōēż 100, `since_id` cursor; returns `hits[]`, `total`, `dropped`, `lastId`). Compiled out in release ŌĆö only active with `-D MULTIANIM_DEV`. 9 regression tests in `DevBridgeTest` covering empty buffer, capture without pause, id increment, `since_id` cursor (including past-latest), `limit`, `clear`, ring-buffer drop behavior, and paused-flag side-effect.
- **`UIInteractiveSource` interface** (`bh.ui.UIInteractiveSource`) ŌĆö abstraction over anything that can feed interactive elements to `UIScreen.addInteractives` / `UIRichInteractiveHelper.register`. Four methods: `getInteractives():Array<MAObject>`, `addRebuildListener(fn)`, `removeRebuildListener(fn)`, `setParameter(name, value)`. Implemented by both `BuilderResult` (runtime path) and every `@:manim` codegen-generated instance class, so both paths are interchangeable at call sites. `UIScreen.addInteractives` and `UIRichInteractiveHelper.register`/`resync`/`bind` now accept `UIInteractiveSource` instead of `BuilderResult` ŌĆö existing `BuilderResult` calls still work via polymorphism.
- **Codegen instances expose their interactives** ŌĆö `@:manim` instance classes now generate `getInteractives():Array<MAObject>` (walks the instance's scene graph via `ProgrammableBuilder.collectInteractives`), `addRebuildListener`/`removeRebuildListener` (append/remove from internal `_rebuildListeners` array), a private `_fireRebuildListeners()` helper fired at the end of every typed setter (after `_applyVisibility` / `_updateExpressions`), and a generic `setParameter(name:String, value:Dynamic)` dispatcher that routes to the right typed setter (`setStatus` etc.). The enum dispatcher accepts either an Int index or a String enum value (e.g. `instance.setParameter("status", "hover")` looks up `"hover"` in the param's enum values at codegen time). Instance classes now declare `implements bh.ui.UIInteractiveSource` so `screen.addInteractives(codegenInstance, prefix)` works symmetrically to the runtime path. Switch arm flips inside codegen programmables auto-resync the screen's wrapper map through the same rebuild listener mechanism used by the runtime path. 7 integration tests in `UIRichInteractiveHelperTest` covering `getInteractives`, direct screen wiring, string-dispatched `setParameter`, full `UIRichInteractiveHelper` end-to-end cycle, autoStatus end-to-end, and `@switch` arm resync on `setMode()`.
- **`BuilderResult.addRebuildListener(fn)` / `removeRebuildListener(fn)`** ŌĆö register a callback that fires once per `applyUpdates()` cycle in which any parameter changed. Use to resync external state (e.g. screen-side `UIInteractiveWrapper` maps) after a `@switch` arm flip or param-dependent `repeatable` count change. The listener fires AFTER all conditional/tracked updates and dynamic ref propagation, so the BuilderResult collections are coherent when it runs. Re-entrant `setParameter()` from inside a listener is supported (enters a fresh applyUpdates cycle). No-op on non-incremental results (static snapshots never rebuild), so callers can install listeners without branching on incremental mode.
- **`UIRichInteractiveHelper.resync(result, ?prefix, ?metadataKey)`** ŌĆö resync bindings against the BuilderResult's current `interactives` list. Removes bindings whose ids no longer exist (filtered by prefix), adds bindings for new ids, and *preserves* existing bindings' state (does not reset `currentState` to Normal). Use after a SWITCH arm flip / REPEAT rebuild changes which interactives are present. The screen's auto-status helper is now resynced automatically (see "Fixed" below).
- **`@switch` block** ŌĆö `@switch(param) { value: ...; default: ... }` groups multiple conditions on a single parameter. Supports single-value arms (`idle: ...`), multi-value with pipe (`hover | pressed: ...`), comparisons (`<= 10:`, `> 90:`), ranges (`0..100:`), `default:` fallback, and block arms (`value { ... }`) for multiple elements per case. Block arms can contain nested `@()` conditionals. Implemented as `SWITCH` NodeType with O(1) Haxe `switch` dispatch in codegen for pure-enum arms. No unnecessary scene graph wrappers ŌĆö single-child arms toggle visibility directly on the child.
- **Conditional blocks** ŌĆö `@(cond) { ... }`, `@else { ... }`, `@else(cond) { ... }`, `@default { ... }` ŌĆö group multiple elements under a single condition without repeating the `@` prefix on each element. Wraps children in a POINT node. Supports nesting (block-in-block).
- **`@any`/`@all` conditional keywords** ŌĆö `@any(p1=>v1, p2=>v2)` matches when ANY listed condition matches (OR logic). `@all(p1=>v1, p2=>v2)` matches when ALL listed conditions match (AND logic, same as `@()`/`@if()`). Unlisted parameters are ignored in both modes. Stacked conditionals (`@(a=>1) @(b=>2)`) now produce a clear error directing users to `@all()` or `@any()`.

- **Tile source `center()`/`pivot()` modifiers** ŌĆö wrap any tile source to set the tile's origin point. `center(source)` is shorthand for `pivot(0.5, 0.5, source)`. `pivot(x, y, source)` sets an arbitrary pivot using 0ŌĆō1 ratio coordinates. Works everywhere tile sources are used: `bitmap()`, particle `tiles:`, etc. When used inside `bitmap()`, overrides the bitmap's hAlign/vAlign. Implemented as `TSPivot` enum variant in `TileSource`, handled in builder, codegen, and all three bitmap creation paths.

- **Dynamic programmable references in `dynamicRef`** ŌĆö `dynamicRef($template, val=>$hp)` where `$template` is a `string` or enum parameter of the enclosing programmable. The resolved parameter value names the target programmable. When the template parameter changes, the old visual is torn down and the new programmable is fully rebuilt. Forwarded parameters (`val=>$hp`) propagate incrementally without rebuild. Backward compatible: `$progName` where `progName` is not a parameter falls back to literal name. Strict validation: forwarded parameters must exist on the target programmable (RTE if missing, allowed if target has a default). Works in both builder (via `IncrementalUpdateContext.trackDynamicName`) and codegen (via `generateDynamicNameRefCreate` with container/name/result fields).

- **`UIEventPriority` constants and overlay priority** ŌĆö `UIEventPriority` class with `Content` (0), `Overlay` (100), `Modal` (200) tiers for event priority. Dropdown (`UIStandardMultiAnimDropdown`) now implements `UIElementPriority` ŌĆö sets `Overlay` when panel opens, `Content` when closed. `UIPanelHelper` sets `Overlay` priority on all panel interactives. `UIInteractiveWrapper.eventPriority` is now publicly writable for programmatic override. Fixes the dropdown click-through bug where background interactives received events through open overlays.
- **`ScreenShakeHelper`** ŌĆö lightweight additive screen shake helper for impact feedback. Supports basic shake with linear decay, directional shake (horizontal recoil, vertical landing), and custom decay curves. Multiple concurrent shakes stack naturally. Non-destructive: applies offsets as per-frame deltas relative to the previous frame, so gameplay can freely move the target (camera scroll, layout, animation) without the shake fighting back to a captured baseline. Uniform jitter via `hxd.Rand`.
- **Particle `emitFilter`** ŌĆö optional callback on `ParticleGroup` to filter particles by spawn position. Called after world position is computed; return `false` to discard the particle. Works for both relative and non-relative groups.
- **Particle `externallyDriven`** ŌĆö when `externallyDriven: true`, particles do not auto-update from the render loop. Game code drives the simulation via `group.advanceTime(dt)` or `particles.advanceTime(dt)`. Follows the same pattern as `AnimationSM.externallyDriven`. Both group-level time (shutdown, attached paths, spawn curves) and per-particle physics are suppressed until manually advanced.
- **Grid `sceneToHex()`** ŌĆö convert scene coordinates to hex grid coordinates. Returns the nearest hex `CellCoord` regardless of whether a cell exists. Works for hex grids only (returns null for rect).
- **`.anim` `flipX` / `flipY`** ŌĆö flip all frames of an animation horizontally/vertically *in place*. The sprite keeps the same untrimmed screen footprint regardless of where the pivot sits within it (tiles are mirrored around the untrimmed center, trim offsets are recomputed, extrapoints auto-mirror around the center). Usable at the file level (`flipX: yes` before any animation), inside `animation { ... }` blocks, and in the `anim name(flipX:yes): "sheet"` shorthand. Animation-level setting overrides file-level default. Shared atlas tiles are cloned before flipping to avoid cross-instance mutation.
- **Particle `shutdown` block and API** ŌĆö graceful particle stop for looping emitters. `shutdown: { duration: 1.0, curve: easeOutQuad, alphaCurve: ..., sizeCurve: ..., speedCurve: ... }` configures wind-down behavior in `.manim`. At runtime, `particles.shutdown()` or `group.shutdown(?duration, ?curve)` starts a timed shutdown that curves particle count (selectively stops recycling dead particles) and optionally applies global alpha/size/speed multipliers. All curves use progress convention (`mult = 1.0 - curve(t)`). Query state via `isShuttingDown()` and `getShutdownRate()`. `emitBurstAt()` still works during shutdown. Existing `onEnd()` fires when the last particle dies.

- **`@switch` incremental mode** ŌĆö `setParameter()` on a `@switch`-controlled parameter now triggers full arm rebuild (teardown + rebuild). Arm content is built non-incrementally; all param refs across all arms are collected so that changing any referenced param (switch param, inner conditionals, repeatable counts, `$param` expressions, positions) triggers the rebuild. Supports nested `@switch`, `@()` conditionals, and repeatables inside arms.
- **`@switch` codegen lazy rebuild** ŌĆö codegen no longer pre-builds all arms and toggles visibility. Instead, uses ordinal-based arm lookup (`switchCounter` at compile time, DFS walk at runtime) to rebuild only the active arm via `ProgrammableBuilder.rebuildSwitchArm()`. Always rebuilds on any parameter change, forwarding current param values to the builder for correct expression evaluation.
- **`@switch` duplicate default rejection** ŌĆö parser now rejects `@switch` blocks with multiple `default` arms at parse time.

### Changed
- **BREAKING: Strict-D color semantics ŌĆö `0x` literals are Heaps AARRGGBB, preserved verbatim** ŌĆö `addAlphaIfNotPresent` is deleted. `ColorArg` abstract is deleted; `PPTColor` storage is a plain `Int`. The parser bakes alpha only for CSS `#` forms (3/6-digit shorthand); `0x...` literals pass through unchanged regardless of length, and every Int-consumer path (`setColor`, `setParameter`, `createFrom`, hot-reload, node tint, graphics shapes, atlas2 autotile, `PixelOutline.inlineColor`, `ReplacePaletteShader`) trusts its input. This fixes the BUG-3 footgun where `setColor(0)` was silently clobbered to `0xFF000000` (opaque black) and `#00000000` / `0x00000000` transparent was unreachable from runtime callers ŌĆö transparent and opaque-black are now distinguishable everywhere. **Migration:** any `.manim` source using `0xRRGGBB` (6-hex Heaps literal) to mean "opaque RRGGBB" must change to `#RRGGBB` or `0xFFRRGGBB`. Automated sed: `sed -i -E 's/0x([0-9A-Fa-f]{6})\b/#\1/g' *.manim`. The 154 occurrences in this repo's test `.manim` files are migrated; cross-repo consumers (`hx-multianim-playground`, `hx-easy-game`) still have unmigrated literals and will render them transparent until their own sed passes run. Any Haxe caller passing `0xFF0000` to a color-accepting API expecting opaque red must change to `0xFFFF0000`. Existing Fixed entries on `setParameter` / `0xRRGGBB` alpha baking (below) documented the old compensation that is now replaced by this simpler "Int is final" rule. The `testCodegenColorParamTransparentFootgun` test is now `testCodegenColorParamTransparentPreserved` and asserts the fixed behavior.
- **`@ifstrict` removed** ŌĆö replaced by `@all()` (AND on listed params, unlisted ignored). The old `@ifstrict` behavior (requiring ALL programmable params to be mentioned) is no longer available.
- **`layout(name)` single-argument form** ŌĆö `repeatable($i, layout("name"))` is now the only accepted form. The legacy two-argument `layout("group", "name")` (where the first argument was a label-only group identifier) is rejected at parse time with a clear error directing users to drop the first argument. All bundled examples, tests, cookbook, and reference docs migrated.
- **Parser cleanup ŌĆö hxparse removal** ŌĆö `MultiAnimParser` is now a thin facade that delegates to `MacroManimParser`. The old hxparse-based lexer/parser, the `MPToken`/`MPKeywords`/`LayoutsParsingState`/`MPInterpolationEnum`/`IdentifierType`/`NumberType`/`ValueType` enums and associated machinery are deleted. A single hand-written recursive-descent parser now covers both compile-time (macro codegen) and runtime paths. Public API (`MultiAnimParser.parseFile()`, `ParametersDefinitions`, `PlaceholderTypes`, etc.) is unchanged.
- **`Particles.addGroup(g)` signature** ŌĆö removed the unused optional `index` parameter (it was never honoured inside the method). Callers passing a second argument must drop it.
- **Hyperlink callback error re-throw in DEV mode** ŌĆö rich text hyperlink callback errors are now re-thrown after tracing when `MULTIANIM_DEV` is defined, making callback bugs visible during development.
- **`@switch` parameter type support** ŌĆö `@switch` single-value arms now route through the same type-aware converter as `@(param => value)`. `@switch` on `color` params accepts `#RRGGBB` literals, on `string` params accepts `"quoted string"` literals, on `bool` params accepts `true`/`false`, and on `int` params accepts bare/negative integers. Previously these either failed to tokenize (strings/colors) or silently produced `CoEnums` string-matches that never matched integer-backed values (bool, color) at runtime. Range/comparison arms (`<= N`, `N..M`, `> N`, etc.) are now rejected at parse time on non-numeric parameters with a clear error. `@switch` on `float`, `tile`, and `flags` parameters is rejected at parse time (use `@(p => bit[N])` for flag tests). Pipe-separated multi-value arms (`v1 | v2 | v3`) are type-aware for discrete non-enum/non-string params (color, int, uint, bool) via a new `CoAnyOf` conditional ŌĆö each arm value is converted through `stringToConditional` and OR'd at match time. Enum/string pipe arms keep the existing `CoEnums` path.
- **`crossfade` transition semantics** ŌĆö `crossfade(duration, ?easing)` is now a sequential blend-through-zero instead of a simultaneous two-sided fade. The outgoing element fades fully to alpha 0 over `duration`, then the incoming element waits and fades from 0 to its target alpha over another `duration`. Total wall-clock duration is `2 ├Ś duration`. Previously both states animated in parallel over a single `duration`, producing a noticeable mid-point alpha spike when old + new were both ~50% visible. Applies to all three transition paths (incremental builder, codegen no-parent, codegen parent-rewrite). Reference image for `95-transition` regenerated.
- **`UIPanelHelper` / `UITooltipHelper` host parameter is now `UIComponentHost`** ŌĆö the constructor `screen` parameter's type changed from the concrete `UIScreenBase` to the `UIComponentHost` interface, completing the same migration previously done on `UICardHandHelper`. `UIComponentHost` gains an `onScreenEvent(event, source)` method (needed by `UIPanelHelper` to push its `EVENT_PANEL_CLOSE` notification). All existing construction sites continue to work unchanged since `UIScreenBase` implements `UIComponentHost`. Enables unit-testing these helpers against a mock host without standing up a full screen.

### Fixed
- **Batched `setParameter` leaked transition specs across unrelated params** ŌĆö `findTransitionSpec()` returned the first match from `changedParams` in `StringMap` iteration order, so a batched update over N unrelated params with an active transition on *any* one of them applied that spec to elements whose own controlling param had no transition. E.g. a `transition { paramA: fade(0.5) }` spec animated `paramB`-controlled elements during a combined `paramA + paramB` update. Fix scopes the spec lookup to the node being toggled: `findTransitionSpec(node)` collects the controlling params via a new `getRelevantParamRefsForNode(node)` (handles `Conditional`, `ConditionalElse` with preceding-sibling chain walk, `ConditionalDefault` with chain walk ŌĆö `collectChainParamRefs` stops at `NoConditional` or an earlier `ConditionalDefault`, mirroring `resolveVisibilityForChildren`), then picks the alphabetically-first controlling param that both has a spec and was changed this cycle. When no controlling param has a spec, returns null (instant toggle). Regression: `testTransitionBatchedUpdatesScopedToParamRefs` in `BuilderUnitTest` builds a programmable with `paramA: fade(0.5)` + a non-transitioned `paramB`, batches both setters, and asserts that only `paramA`-driven elements carry tweens.
- **`@else` / `@default` branches flashed through a full fade-out when their chain decision was unchanged** ŌĆö `IncrementalUpdateContext.applyUpdates` ran a flat `isMatch()` pass over `conditionalEntries` before calling `applyConditionalChains()`. `isMatch()` returns `true` unconditionally for `ConditionalElse` / `ConditionalDefault` (chain semantics only resolve inside `applyConditionalChains`). Pass 1 therefore `addToGraph`-ed the @else wrapper and started a fade-in whenever any param driving the chain's first arm changed; pass 2 then realized the @else was still inactive, cancelled the fade-in, and started a fade-out ŌĆö leaving the element in graph at `savedAlpha` for the fade duration. Visible as a flash of a branch that never should have appeared. Fix removes the redundant pass: `applyConditionalChains` already walks the full tree from `rootNode`, handles `Conditional`/`ConditionalElse`/`ConditionalDefault` correctly via the per-node chain walk, and dispatches to both `conditionalEntries` and `conditionalApplyEntries` via `uniqueNodeName` lookup. `setPresenceWithTransition` and `setPresenceOrMaterialize` now take the `Node` so transition-spec lookup is node-scoped (see sibling entry). Regression: `testTransitionDoesNotFlashInactiveElseBranch` in `BuilderUnitTest` triggers the buggy path (transition on a non-chain param) and asserts no extra children, no extra visible bitmaps, and no tweens on the unchanged-visibility chain.
- **`dynamicRef` parameter propagation cascaded into hidden subtrees** ŌĆö the `dynamicRefBindings` loop in `applyUpdates` gated only on whether a referenced parent param was in `changedParams`, with no visibility check. When a `dynamicRef` sat inside a hidden `@()` branch, parent `setParameter` calls to forwarded params still invoked `childContext.setParameter(...)` on the detached child ŌĆö re-running its `applyUpdates` cycle (expression eval, conditional re-check) and firing any registered `addRebuildListener` callbacks. Wasted work and, for listeners that resync external state, a spurious notification. Fix adds an `object:Null<h2d.Object>` field to each binding (threaded through `trackDynamicRef` from both the initial build at `DYNAMIC_REF` and the `rebuildDynamicNameRef` path) and gates propagation on `isEffectivelyVisible(obj)`, mirroring the policy the `trackedExpressions` loop directly above already uses. Null-object fallback still propagates (matches the tracked-expressions `obj != null && !ŌĆ”` convention) so any caller that omits the new optional arg is unaffected. Regression: `testDynamicRefPropagationSkipsHiddenSubtree` in `BuilderUnitTest` attaches a rebuild listener to the child, hides the wrapper, changes the forwarded param, and asserts the child's listener does not fire.
- **`resolveAsString` RVParenthesis and `resolveAsBool` comparison ops silently swallowed all exceptions** ŌĆö the `try { resolveAsNumber(e); ŌĆ” } catch (_:Dynamic) { return resolveAsString(e); }` pattern at two callsites (the string-interpolation `${expr}` path and the `<`/`>`/`<=`/`>=` numeric-fallback-to-string path in `resolveAsBool`) caught every thrown exception, including real errors like missing references, division by zero, or unsupported expression types. Users got unintended string concat / string comparison instead of the real error. Now catches only `BuilderError` with `code == "not_a_number"` (thrown from the specific sites where a value is well-formed but non-numeric: `RVString` NaN, `RVColor*`, callback returning `CBRString`). All other errors re-throw. `resolveAsBool`'s 4 separate try/catch blocks extracted into a shared `compareNumericOrString` helper with the narrow filter.
- **`generated(color(w, h, #c))` silently dropped explicit #RRGGBBAA alpha** ŌĆö both the runtime (`MultiAnimBuilder.generatePlaceholderBitmap`) and codegen (`ProgrammableCodeGen` `SolidColor`/`Cross`) paths called `h2d.Tile.fromColor(color, w, h)` with no alpha arg. Heaps defaults alpha to 1.0 and masks the top byte of `color`, so parser-baked strict-D values like `#7fdbda33` (ŌåÆ `0x337fdbda`, alpha `0x33`) rendered fully opaque ŌĆö the author's explicit alpha byte was ignored. Same bug hit `.manim` `@final RGBA8 = #FF880080` constants used as `generated(color(...))` args, including those resolved through `$param` refs. Now both paths go through the shared `HeapsUtils.solidTile` helper which unpacks the alpha byte: `top = color >>> 24; alpha = top==0 ? 1.0 : top/255.0; Tile.fromColor(color & 0xFFFFFF, w, h, alpha)`. Three codegen regression tests in `ProgrammableCodeGenTest`: `testCodegenSwitchArmGridContext`, `testCodegenSwitchArmCallbackInheritance`, `testCodegenSlotParamGridContext`.
- **LSP `.anim` context detection in comments/strings** ŌĆö `AnimAnalyzer.detectContext` walked every character of every line counting `{`/`}` and keyword-matched line starts without comment or string awareness. Braces inside `//`/`/* */` comments shifted the nesting depth, and keywords inside strings or comments (e.g. `// playlist disabled`) flipped the completion context to the wrong block, corrupting hover/completion results for any `.anim` file with prose. Lexer now strips line comments, tracks block comments across lines, and skips `"..."` string literals (including `\"` escapes) before counting braces. Keyword detection is suppressed while inside a block comment.
- **Particle `emitFilter` sub-emitter cascade** ŌĆö when `emitFilter` rejected a particle, the particle's `OnBirth` trigger still fired during the next `init()` step and, once its forced-dead lifecycle tripped, its `OnDeath` trigger fired as well, producing an explosive spawn loop at the rejection boundary (every rejected particle spawned a full child burst plus a death burst). `Particle.rejected` now flags filter-rejected particles; `init()` clears it on every fresh spawn and `emitFilter()` sets it when discarding. Both `OnBirth` (delayed spawn step) and `OnDeath` (lifecycle expiry) sub-emitter triggers skip rejected particles.
- **Particle recycle double-init** ŌĆö when a particle's lifecycle expired on a looping group with live headroom, `init()` ran to recycle the particle but `timeNormalized` (computed earlier in the same update step from the pre-init `life`) was still > 1, so the lifecycle branch immediately re-fired `OnDeath` triggers and potentially re-ran the recycle path with stale state. The recycle branch now returns `true` immediately after `init()`, skipping the stale lifecycle check for that frame.
- **Particle shutdown terminal multipliers** ŌĆö when `shutdown()`'s curve reached `rate >= 1.0` and `shutdownTargetCount <= 0`, the shutdown flag was cleared so remaining particles could die naturally, but `shutdownAlphaMult`/`shutdownSizeMult`/`shutdownSpeedMult` still held their terminal (~0) values from the final curve sample. Any live particles finishing their lifetime after shutdown exit rendered at 0 alpha/size/speed instead of their natural values. All three multipliers are now reset to `1.0` at the moment `shutdownActive` flips to false.
- **Particle `removeGroup()` batch leak** ŌĆö `Particles.removeGroup(id)` only removed the group entry from the map and left the underlying `h2d.SpriteBatch` attached to the scene with its particles. The batch and its elements stayed resident and continued rendering until the parent `Particles` object was disposed. Now clears and removes the batch before dropping the map entry.
- **Particle `tiles.length == 0` early return** ŌĆö `init()` previously `return`ed from the middle of the function when a group had no tiles (waiting for async asset load), skipping the remainder of the setup (`p.vx *= speed`, gravity/acceleration/rotation assignment, `emitFilter()` check). First-frame particles from tile-less groups started with unscaled velocity and bypassed `emitFilter`. Now wraps tile selection in a conditional so the rest of init always runs.
- **`ProgrammableBuilder.setFontFromMultiple` empty-fonts crash** ŌĆö when the font fallback array was empty (no best match and no fonts at all ŌĆö e.g. misconfigured rich text), `fonts[fonts.length - 1]` evaluated to `fonts[-1]` which is `null` on HashLink and could crash further downstream. Now retains the original font when the array is empty instead of clobbering it with null.
- **`.anim` conditional duplicate state key** ŌĆö `@(level >= 3, level <= 5, level != 0)` silently overwrote prior entries for the same state name in the conditional map, so only the last comparison for a given state was applied. Parser now rejects duplicate state keys in a single conditional with a clear error.
- **`.anim` negative integer comparison literals** ŌĆö `@(level >= -1)`, `@(xp > -100)`, etc. failed to parse because `expectIdentifier()` didn't accept a leading minus. Comparison conditionals now use `expectSignedIdentifier()`, matching the negative-literal handling already in place for plain value lists.
- **`.anim` parser `throw` ŌåÆ `syntaxError`** ŌĆö several late-phase validation errors (`no playlist for state X`, `animation X not reachable`, `Extra point X in anim Y not reachable`) were raised via raw `throw` statements, bypassing the parser's structured error reporting (file/line/column context). All now route through `syntaxError()` for consistent strict-mode output.
- **`@switch` on `:array` params** ŌĆö `@switch` on an array-typed parameter previously hit the `default:` branch of the type check and silently emitted string arms that could never match. `PPTArray` now surfaces a clear "Array cannot be matched ŌĆö @switch requires a discrete type" error at parse time.
- **`MacroManimParser` iterator error message** ŌĆö the error listing valid iterator types for `repeatable()` / `tilegroup` named the leading option `grid`, but the parser rejects `grid` and accepts `step`. Corrected to list `step, layout, array, range, stateanim, tiles`.
- **`ManimKeywordInfo` missing `SWITCH` entry** ŌĆö the element-ctor and node-type lists used by the LSP's exhaustive-switch compile-time guard did not include `SWITCH`, so any addition to those enums that touched `SWITCH` would have failed the exhaustive check silently. Added `SWITCH` to both lists.
- **`UICardHandHelper` arrow ignoring `canPlayCard`** ŌĆö while dragging a card into a targeting zone, the arrow's valid/invalid color (green vs red) was based solely on whether the cursor hovered a registered target interactive. A `canPlayCard` veto (e.g. "this shuttle can't board that tile") silently refused the drop but the arrow still lit green during the drag, giving the player no feedback. `updateCardDrag()` now calls `canPlayCard(cardId, currentResult)` each frame and forces `targeting.forceValid` to match, so invalid drops flash red before release.
